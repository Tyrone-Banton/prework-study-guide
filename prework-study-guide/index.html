Prework Study Guide

HTML

  • The DOCTYPE html declaration at the very top, is letting the browser know to expect HTML on the page
  • The HTML element represents the start of my HTML document.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS rule: A selector defines the element or attributes to which declarations apply
  • Declarations contain two components: CSS property to apply and the value of property. For example the declarations colour: blue; includes the CSS property colour and the value for that property blue.
  • A padding indicates how much space we want around the content inside an element.
  • A margin indicates how much space we want around the outside of an element.
  • touch style.css: creates a file containing your CSS style sheet in GitBash
  • Use: link rel="stylesheet" href="./assets/style.css" to link HTML to CSS file

Git

  • git push origin "working branch name"
  • git commit -m "message example": commits the changed flies once staged, into our repository with a short description
  • git add -A: to add the changed flies in our working branch to the staged area of GitHub
  • git checkout main: to switch to the main branch
  • git pull origin main: ensures our local branch is in-sync with the remote branch in GitHub
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
